404  // A note about the organization:
405  //
406  // FEC is performed across the contents of the orig_cache_ which consists of
407  // a number of slots. Each slot may hold a single original packet, part of
408  // an original packet, or multiple original packets depending on the
409  // relationship of the maxchunksz to the observed packet sizes. Each
410  // original packet within an encoding group is assigned a unique
411  // pktID. Within a group, pktIDs start with 0 and increment by one as each
412  // additional packet is received and processed.
413  //
414  // When a packet is split across multiple slots, only contents from that
415  // packet are used to fill each slot -- i.e., a slot will never contain
416  // fragments from several packets. In this instance, each partial packet is
417  // referred to as a fragment, and is assigned a fragID to assist in the
418  // reassembly
419  //
420  // When multiple packets are contained within a slot, only complete packets
421  // are contained within that slot -- i.e., a slot will never contain
422  // fragments from a packet along with partial or complete portions of any
423  // other packet In this instance, each multiple packet slot is considered to
424  // be a blob and is (implicitly) assigned a blobID to assist in the
425  // reconstitution
426  //
427  // To simplify the data structures, we use the term "chunk" to refer to both
428  // fragments and blobs, and we use a chunkTrailer to assist in the
429  // reassembly/reconstitution of both fragmented packets and multi-packet
430  // constructs. When the contents of a slot contain a blob, the haveBlob flag
431  // is set to true, the pktID refers to the first packet within the chunk,
432  // and nChunks contains the number of packets within the chunk (blob) The
433  // chunkID field is not used.
434  //
435  // When the contents of a slot contain a fragment, the haveBlob flag is set
436  // to false, the pktID refers to the packet ID across all frgaments, nChunks
437  // describes the number of fragments the original packet is spread across,
438  // and the chunkID describes the position within original packet the current
439  // chunk represents (i.e., which of the nChunks this fragment represents)
