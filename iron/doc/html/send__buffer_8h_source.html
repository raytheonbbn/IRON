312  // The send buffer and retransmission list are depicted below:
313  //
314  // snd_una_ snd_nxt_ tail_ uwe
315  // | | | |
316  // v v v v
317  // +-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+
318  // | PI* | PI* | PI* | PI* | PI* | PI* | PI* | PI* | | |
319  // +-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+
320  // | | | |
321  // | | ---| ---------|
322  // ---| ---| | |
323  // | | | |
324  // v v v v
325  // +-----+-----+-----+-----+
326  // | PI* | PI* | PI* | PI* |
327  // +-----+-----+-----+-----+
328  // ^ ^
329  // | |
330  // rexmit_head_ rexmit_tail_
331  //
332  // Packets are placed into the send buffer as they are received.
333  //
334  // If a packet has a retransmission time set, it is placed into the
335  // retransmission list. NOTE: the retransmission list simply contains
336  // pointers to the real packets in the send buffer, i.e., it is a "second
337  // set of books" that enables for fast access to packets that need to be
338  // retransmitted. The retransmission list does not duplicate packets.
339  //
340  // Packets are removed from the send buffer as TCP ACKs are received. If a
341  // packet is removed that has a corresponding entry in the retransmission
342  // list, the packet is also removed from the retransmission
343  // list. Additionally, the Packets can be removed from the send buffer as
344  // SACK Plugs are received. Once a Packet is removed from the send buffer,
345  // it is never added back.
