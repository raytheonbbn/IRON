Detailed Description

A wrapper for adding pacing to congestion control algorithms.

Note that this class is not thread-safe.

Time PacingSender::TimeUntilSend ( const iron::Time &  now)
virtual

Calculate the time of the next data packet transmission.

The method is used to implement send pacing of data packets. If the returned time is zero, then a transmission can occur immediately. Otherwise, the next transmission must wait for the returned time to elapse first. This method will never return an infinite time.

This method should always be called for new data packets, and should only be called for non-RTO timeout retransmitted data packets if UseRexmitPacing() returns true.

Parameters
nowThe current time.
Returns
The amount of time until the next send can occur.

void PacingSender::UpdateCounts ( int32_t  pif_adj,
int64_t  bif_adj,
int64_t  pipe_adj 
)
virtual

Called when the packet counts must be adjusted.

The packets in flight and bytes in flight counts include all unACKed packets. The pipe size count includes all packets not ACKed, and adds the packet size once if the packet is not lost, and adds the packet size once if the packet has been retransmitted (so a packet that is not lost and retransmitted will be counted twice). See RFC 6675, section 2.

This method must be called once after the all of the OnPacketLost(), OnPacketAcked(), or OnPacketSent() methods are called for handling an ACK packet or sending a new data packet.

Parameters
pif_adjThe adjustment to the number of packets in flight. Can be positive, zero, or negative.
bif_adjThe adjustment to the number of bytes in flight. Can be positive, zero, or negative.
pipe_adjThe adjustment to the pipe size, in bytes. Can be positive, zero, or negative.
