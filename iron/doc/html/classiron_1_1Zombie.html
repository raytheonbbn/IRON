Detailed Description

Class of static utility functions for creating zombie packets.

A zombie packet is a packet sent around using IRON backpressure forwarding purely as a graceful backpressure signalizing mechanism. That is, it will contribute to the gradients (and live in the queues) at the IRON nodes, but it will never be sent to an application. If zombie compression is enabled, these zombies will also not take up as much space on the links as a real packet of the same (virtual) size.

Generate a new zombie packet from scratch.

This function generates an IP packet (that is not UDP or TCP) that will function simply as a space-hog in the BPF queues and gradients. If zombie compression is enabled, it will not hog (much) space on the links.

void Zombie::ZombifyExistingPacket ( Packet *  pkt)
static

Change a data packet into a zombie packet.

This function simply marks the packet as lowest forwarding priority (i.e., least restrictive latency requirement), and unsets any time to go, so that the packet can continue to be processed and forwarded in the IRON network until it reaches its destination IRON node. This will also compress the packet if zombie compression is enabled.

Parameters
pktThe data packet that should be turned into a zombie.
